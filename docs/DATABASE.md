# Database Schema & Encryption

Thalavedana uses `node:sqlite` for local database management.

## Table Schemas

### 1. `settings`
Stores configuration keys and encrypted secrets.

| Column | Type | Description |
|---|---|---|
| `key` | TEXT (PK) | Configuration key name (e.g. `geminiApiKey`) |
| `value` | TEXT | Encrypted base64 payload or raw string |
| `is_encrypted` | INTEGER | Flag (`1` if encrypted via keyring, `0` otherwise) |

### 2. `repositories`
Tracks repositories scanned by the Git CLI scraper.

| Column | Type | Description |
|---|---|---|
| `id` | INTEGER (PK) | Auto-incrementing identifier |
| `path` | TEXT | Absolute path to the Git repository (Unique) |
| `name` | TEXT | Human-readable directory name |
| `created_at` | DATETIME | Timestamp of entry creation |

### 3. `reports`
Logs historical work report outputs, Excel logs, and Gmail progress.

| Column | Type | Description |
|---|---|---|
| `id` | INTEGER (PK) | Auto-incrementing identifier |
| `report_date` | TEXT | Calendar date of report in `YYYY-MM-DD` (Unique) |
| `commit_data` | TEXT | JSON string representing the scanned git commits |
| `report_content` | TEXT | Markdown daily report generated by Gemini |
| `email_content` | TEXT | JSON string containing email subject and body |
| `excel_status` | TEXT | `'pending' \| 'updated' \| 'failed'` |
| `email_status` | TEXT | `'pending' \| 'sent' \| 'failed'` |
| `error_message` | TEXT | Error text if Excel or Gmail steps failed |
| `created_at` | DATETIME | Timestamp of entry creation |

### 4. `logs`
Captures system operations and diagnostic messages.

| Column | Type | Description |
|---|---|---|
| `id` | INTEGER (PK) | Auto-incrementing identifier |
| `timestamp` | DATETIME | Time of the log entry |
| `level` | TEXT | `'INFO' \| 'WARN' \| 'ERROR'` |
| `category` | TEXT | System module (e.g. `'GMAIL'`, `'GIT'`, `'SCHEDULER'`) |
| `message` | TEXT | Detailed log message |

## Secret Encryption Mechanism

When writing keys designated as sensitive (defined in `SENSITIVE_KEYS` array in `database/index.ts`):
1. **Encrypt**:
   - `safeStorage.encryptString(plainText)` encrypts the value using OS keyrings.
   - The encrypted buffer is encoded into a `base64` string and written to `settings.value`.
   - `settings.is_encrypted` is set to `1`.
2. **Decrypt**:
   - Reads `value` and `is_encrypted`.
   - If `is_encrypted` is `1` and encryption is available, decodes `base64` back to a buffer and calls `safeStorage.decryptString(buffer)`.
   - If decryption fails (e.g. keyring locked), it falls back to empty or logs an error.
